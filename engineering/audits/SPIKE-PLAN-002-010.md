# Phase-0 Spike Plan — SPIKE-002…010 (experiments, not documents)

> Each spike is an **experiment**: hypothesis → method → pass/fail → what it decides. Results land in the [Evidence Register](EVIDENCE-REGISTER.md) (and, where they change design, an RFC). Marked **[HW]** = requires physical hardware (founder/engineer executes; protocols below make the hardware time efficient) or **[DESK]** = executable without devices. SPIKE-001 ✅ done (platform-truth research).
>
> **Equipment list (one purchase, serves all HW spikes):** 1× Apple Watch **cellular** Series 9/10-class + 1× Series 4/5 or SE (old-hardware floor) + 1× test iPhone + 1× eSIM/plan + a second phone for responder-side testing. Prereq for several: **A1 entitlements filed** (fall detection ~days; critical alerts days–weeks).

| Spike | Question (hypothesis) | Method | Pass / Fail | Decides |
|---|---|---|---|---|
| **SPIKE-002 [HW]** Family Setup validation | *H: fall-detection API + an independent RAXHA watch app work on a managed (no-iPhone) LTE watch* | Configure Family Setup on cellular watch; install minimal independent app with fall entitlement; trigger simulated fall (Apple's developer simulation); observe native UI → `didDetect` delivery; test HealthKit reads (E-08) + LTE event upload w/o phone | PASS = event+resolution delivered & uploadable ⇒ **watch-only v1 shape viable** / FAIL ⇒ iPhone-paired v1 stands | **RFC-004, RFC-005** — the most decision-heavy unknown (E-07) |
| **SPIKE-003 [HW]** Battery profiling | *H: coordinator-mode (event-driven) fits all-day battery on Series-4-class; armed-mode (workout session) costs X%/hr* | Instrumented day-long runs: idle coordinator vs armed workout-session mode, both watch generations; log %/hour | Budget table with real numbers (replaces TD-7 bench estimates) | E-14, hardware floor, PRD battery claims, SPIKE-009 overlap |
| **SPIKE-004 [HW]** Background execution timing | *H: post-event background time suffices for confirm+upload (<Xs) across app states* | Fire simulated falls with app foreground / background / terminated / Low Power Mode; measure `didDetect`→upload-complete latency distribution | p95 latency table per state; FAIL if terminated-state delivery unreliable | E-16, RFC-001 detail, latency budget (SRS-1002 reality) |
| **SPIKE-005 [HW]** Reboot behavior | *H: journal + relaunch recover an in-flight incident on watch & phone reboot* | Start synthetic incident → force reboot at COUNTDOWN/ALERTING → observe relaunch, journal recovery, resume-vs-dead-man behavior | PASS = no lost/duplicated incident (VV-106/201 pre-validation) | E-15, ADR-105 reality |
| **SPIKE-006 [HW]** Network-loss behavior | *H: offline decision + queue + SMS-fallback path works; recovery syncs idempotently* | Airplane-mode matrices (watch-only offline / phone offline / both); run incident; restore; verify exactly-once-effect | PASS = zero loss, zero duplicates | E-13 partial, VV-203/206 pre-validation |
| **SPIKE-007 [HW]** Escalation deliverability | *H: registered SMS + branded voice reach a silenced 3 a.m. phone within rung timeouts* | After **A2 registrations**: full ladder test against carrier-diverse responder phones, DND on; measure delivery + "Scam Likely" labeling | Deliverability table per carrier/rung; FAIL rungs get vendor changes | E-13, H5, ladder timeouts |
| **SPIKE-008 [HW, needs entitlement]** Critical alerts | *H: critical alerts pierce silent/DND/Focus on responder devices as documented* | With granted entitlement: alert matrix across Focus modes/volumes | PASS = audible alert in all modes | E-03, IF-HUM-04 |
| **SPIKE-009 [HW]** Workout-session endurance | *H: armed-mode (continuous session + high-rate sensing) survives ≥N hours within battery/thermal limits* | Multi-hour armed sessions on both watch generations; log battery, thermal, session terminations | Real armed-mode duration ceiling → PRD armed-mode claims | E-05 carve-out sizing, RFC-001 |
| **SPIKE-010 [DESK ✅ started]** Replay corpus | *H: a defensible v1 corpus can be assembled from public data + own recordings* | **Findings (2026-07-14):** [SisFall](https://www.mdpi.com/1424-8220/17/1/198) ✅ open — 15 fall types but performed by *young adults*, belt-worn (use for ADL/false-positive classes + waveform shapes; never claim elderly validation from it — VA-02). MobiAct 🟡 20 Hz, too coarse — deprioritized. [**FARSEEING**](https://link.springer.com/article/10.1186/s11556-016-0168-9) — **208 verified real-world elderly falls, shared on request** → **Action A4: founder emails the consortium** (the only real-elderly ground truth in existence). Remainder: own Apple-Watch ADL recordings (SPIKE-002/003 sessions double as corpus collection) | Corpus manifest with per-source provenance + honest-use labels | VA-01/02, VV-103/104 inputs, H6 cold-start gates |
